# Актуалізований аудит матеріалів AURA/Ecliptix для докторської роботи

## 1. Поточна рамка

Найсильніша докторська рамка зараз така:

**Методологія та інформаційна технологія захисту контекстної приватності особи в умовах інференційних загроз цифрового середовища.**

У цій рамці AURA/Ecliptix не подається як окремий застосунок. Це програмне втілення ширшої наукової ідеї: середовища виконання політик захисту контекстної приватності, яке працює без серверного доступу до відкритого тексту повідомлень.

## 2. Основні складники доказової бази

| Складник | Роль у докторській | Коментар |
|---|---|---|
| `aura-security-opaque-rs` | метод постквантово-посиленої OPAQUE-автентифікації | найзріліший криптографічний матеріал |
| `aura-protected-protocol-rs` | метод гібридного постквантового наскрізного шифрування | потребує узгодженого формулювання меж формальної перевірки |
| `ecliptix-aura` | середовище виконання політик контекстної безпеки | головна реалізаційна опора AURA |
| `aura-platform` | серверна частина інформаційної технології | доцільно брати як основну серверну реалізацію |
| `aura-messenger-ios` | мобільний застосунок та інтеграційна перевірка | корисний як доказ переносності й інтеграції |
| `ecliptix-auth-relay` | історична або допоміжна гілка каналу нагляду | не варто робити головним джерелом, якщо `aura-platform` покриває сценарій |

## 3. Наукове позиціонування

Роботу не треба позиціонувати як “ми побудували захищений застосунок”. Захищений застосунок є середовищем перевірки, а не науковою новизною сам по собі.

Захищене формулювання:

> запропоновано методологію, методи та інформаційну технологію захисту контекстної приватності особи, у якій постквантово-посилена автентифікація, гібридне наскрізне шифрування, локальне виконання політик контекстної безпеки та серверно-сліпий канал нагляду поєднані в єдиний захисний контур із явними межами тверджень.

## 4. Наукова новизна

1. Архітектура захисту контекстної приватності на основі середовища виконання політик AURA.
2. Метод контекстної оцінки інформаційного ризику з урахуванням стану взаємодії.
3. Метод прийняття захисних рішень з урахуванням політик.
4. Метод постквантово-посиленої OPAQUE-автентифікації.
5. Метод гібридного постквантового наскрізного шифрування.
6. Метод серверно-сліпого каналу нагляду.
7. Модель меж компрометації та методологія приватного аудиту.
8. Інформаційна технологія AURA/Ecliptix з відтворюваною перевіркою.

## 5. Ризики, які треба контролювати

### Ризик 1. Надмірне твердження про психологічний захист

Не стверджувати клінічну або психологічну валідність. Маніпуляція, грумінг, примус і самопошкодження мають подаватися як формалізовані сигнали ризику, а не як діагнози.

### Ризик 2. Змішування криптографії та контекстної безпеки

Криптографія створює умови приватного виконання політик, але не доводить коректність самих захисних рішень. Це треба явно розмежувати в розділах 2, 5, 6 і 8.

### Ризик 3. Нечітка межа кінцевого пристрою

Усі твердження про контекстну безпеку мають містити припущення про цілісність кінцевого пристрою. Повністю скомпрометований пристрій є межею застосування, а не властивістю, яку робота має довести.

### Ризик 4. Неповна відтворюваність

Для кожного методу потрібні:

- зафіксована версія репозиторію;
- команда перевірки;
- очікуваний результат;
- межі застосування результату;
- короткий опис того, що саме підтверджує перевірка.

## 6. Рекомендована структура

1. Аналіз стану проблеми.
2. Концепція інформаційної технології на основі середовища виконання політик AURA.
3. Метод контекстної оцінки інформаційного ризику.
4. Метод прийняття захисних рішень з урахуванням політик.
5. Метод постквантово-посиленої OPAQUE-автентифікації.
6. Метод гібридного постквантового наскрізного шифрування.
7. Метод серверно-сліпого каналу нагляду.
8. Інформаційна технологія AURA/Ecliptix та експериментальна перевірка.

## 7. Висновок аудиту

Матеріал достатній для докторської рамки за умови, що робота буде подана не як опис застосунку, а як методологія та інформаційна технологія захисту контекстної приватності. Найсильніший внесок полягає в поєднанні контекстної оцінки ризику, локального виконання політик, постквантово-посиленого криптографічного контуру та серверно-сліпого нагляду в єдину архітектуру з явними межами доказовості.
