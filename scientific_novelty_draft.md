# Чернетка наукової новизни

Робоча тема:

**Методи та інформаційна технологія постквантово-посиленої автентифікації, захищеного обміну повідомленнями та приватної контекстної безпеки в месенджерах**

Наукова новизна одержаних результатів полягає в такому.

## 1. Цілісна архітектура захищеного месенджера

**Вперше запропоновано цілісну архітектуру захищеного месенджера,** у якій постквантово-посилена парольна автентифікація, гібридне наскрізне шифрування, контекстна безпека повідомлень та server-blind supervision relay розглядаються не як ізольовані модулі, а як взаємопов'язані рівні однієї інформаційної технології з єдиною моделлю загроз, межами доведення та межами витоку даних.

Відмінність від наявних підходів полягає в тому, що безпека акаунта, безпека сесійного ключа, приватність контенту та механізми safety-review не вимагають перенесення plaintext moderation на сервер.

## 2. Метод постквантово-посиленої OPAQUE-автентифікації

**Вперше для прикладної архітектури захищеного месенджера розроблено метод постквантово-посиленої augmented-PAKE автентифікації,** який поєднує асиметрію OPAQUE, класичну 4DH-компоненту та ML-KEM-768 у transcript-bound гібридному derivation path.

Метод зберігає властивість, що пароль або password-equivalent матеріал не передається серверу, а межа offline-dictionary атаки формулюється явно через компрометацію бази даних, захист `oprf_seed`, Argon2id та квантову модель adversary.

## 3. Уточнена модель компрометації для password-based PQ transition

**Удосконалено модель аналізу безпеки password-based протоколів у перехідному постквантовому середовищі** шляхом явного розмежування:

- компрометації бази даних;
- компрометації root OPRF secret;
- записаних login transcript-ів;
- квантового відновлення класичного DH-матеріалу;
- ролі KSF як останнього бар'єра для password guessing.

Це дозволяє чесно формулювати, які властивості підсилюються ML-KEM, а які залишаються класично або пароль-залежно обмеженими.

## 4. Метод гібридного постквантового захисту 1:1 повідомлень

**Удосконалено метод побудови наскрізно захищеного 1:1 обміну повідомленнями** через включення ML-KEM-768 не лише в початковий handshake, а й у ratchet-level оновлення ключового матеріалу разом із класичною X25519-компонентою.

На відміну від конструкцій, де постквантовий матеріал використовується тільки на старті сесії, запропонований підхід підтримує hybrid recovery shape після зміни напрямку ратчета та зменшує залежність довгоживучої конфіденційності від одного початкового секрету.

## 5. Метод захисту metadata-bound message flow

**Удосконалено метод захисту службових даних у захищеному messenger protocol** шляхом прив'язки metadata encryption, replay/rollback counters, skipped-key limits та message authentication до єдиного stateful protocol boundary.

Практична новизна полягає в тому, що metadata handling розглядається як частина криптографічного протоколу, а не як другорядна transport-властивість, що дозволяє зменшити ризик протокольного drift між специфікацією, формальною моделлю та реалізацією.

## 6. Метод messenger-native контекстної безпеки

**Вперше запропоновано метод messenger-native контекстної безпеки,** який оцінює ризики не лише за окремим текстом повідомлення, а за станом контакту, історією розмови, часовою динамікою, relationship context, latent manipulation states та продуктовою дією.

Метод відрізняється від generic classifier-підходів тим, що результатом є не тільки label або score, а policy-aware decision surface для реального месенджера: allow, warn, escalate, quarantine, block, guardian/review action.

## 7. Privacy-safe audit and release-gated evaluation

**Дістала подальшого розвитку методологія оцінювання safety-компонентів у приватних месенджерах** через поєднання curated/synthetic messenger-like corpora, support-aware release gates, gold/mixed evaluation slices, dataset governance та privacy-safe audit records.

Наукова цінність полягає в тому, що якість safety runtime оцінюється не тільки aggregate accuracy, а через відтворюваний release decision framework з явними статусами `PASS`, `FAIL`, `INSUFFICIENT_SUPPORT` і `BLOCKED`.

## 8. Метод server-blind supervision relay

**Вперше для E2EE-месенджера запропоновано метод server-blind supervision relay,** у якому child-controlled consent, активний parent-child binding, per-device pair-key provisioning та opaque payload routing дозволяють передавати safety-сигнали guardian endpoint-у без розкриття їхнього plaintext серверу.

Ключова відмінність полягає в тому, що initial relay pair key генерується на клієнті, sealed per recipient device через наявний E2E key-package шлях, а сервер виконує лише авторизацію, rate limiting, зберігання ciphertext, доставку, ACK, revoke та purge lifecycle.

## Коментар щодо кількості

Для фінальної докторської краще залишити **7 пунктів**, якщо треба компактна академічна форма. Тоді пункти 4 і 5 можна об'єднати в один:

**Метод гібридного постквантового протоколу обміну повідомленнями з metadata-bound state protection.**

Якщо комісія очікує ширше розкриття системної новизни, краще залишити **8 пунктів**, але стежити, щоб пункти 4 і 5 не виглядали як штучне дроблення одного результату.

## Найбільш захищена фінальна комбінація

1. Цілісна архітектура захищеного месенджера.
2. Hybrid PQ-OPAQUE authentication method.
3. Уточнена модель compromise boundaries для password-based PQ transition.
4. Hybrid PQ E2EE 1:1 messaging method.
5. Metadata-bound protocol state protection.
6. Messenger-native contextual safety method.
7. Privacy-safe release/evaluation methodology.
8. Server-blind supervision relay.

Ця вісімка виглядає логічно: 1 є системною новизною, 2-6 є методами, 7 є методологією оцінювання, 8 є окремою privacy-preserving data-plane конструкцією.
